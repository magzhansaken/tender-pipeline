# TenderView

**Автоматический поиск, оценка и публикация тендеров госзакупок Казахстана** (goszakup.gov.kz) — с уже найденной ценой поставщика и посчитанной маржой по каждому лоту.

> **В чём суть.** Конкуренты показывают *список тендеров* — клиент сам ищет товар, сам считает прибыль. TenderView показывает **готовую прибыль**: по каждому лоту уже найден товар на маркетплейсе, подставлена цена закупки и рассчитана маржа. Клиент сразу видит, на чём и сколько можно заработать, и забирает готовые сделки.

Стек: **Python · FastAPI · PostgreSQL · Ollama (LLM) · Docker · Caddy**. Фронтенд — без фреймворков (ванильный JS, рукописные SVG-графики).

## Что это делает

1. **Собирает** тендеры (лоты) с goszakup.gov.kz, включая техническое задание (ТЗ) из вложений.
2. **Нормализует** требования из ТЗ через локальную LLM (Ollama) — извлекает характеристики и формирует поисковый запрос.
3. **Ищет** соответствующий товар на маркетплейсах (через DuckDuckGo) и **проверяет** соответствие повторным проходом LLM.
4. **Достаёт** реальную цену поставщика (Kaspi, Satu, Chipdip, Otvertka напрямую в тенге; Wildberries и Alibaba — как ориентир, с пересчётом по курсу Нацбанка РК).
5. **Считает** маржу и **публикует** результат на читаемую витрину.
6. **Продаёт** доступ: цена и маржа открыты на платных тарифах, на демо — скрыты.

Цель — до 1000 клиентов (частные поставщики, ИП, отделы закупок), которым сервис экономит главное: время на поиск товара и расчёт выгоды.

## Как это работает (поток данных)

```
goszakup.gov.kz
      │  (1) daily_sync.py  — сверка с goszakup, новые лоты + синхронизация статусов
      ▼
   tenders (БД, stage='collected')
      │  (2) process_specs.py  — Ollama извлекает характеристики + поисковый запрос
      ▼
   stage='parsed'
      │  (3) search_verify.py  — поиск на площадках (DuckDuckGo) + сверка через Ollama
      │       ├─ сохраняет НЕСКОЛЬКО кандидатов (все площадки, где найден товар)
      │       └─ price_fetch.py — цена с лёгких KZ-площадок (Kaspi/Satu/Chipdip/Otvertka)
      ▼
   stage='searched', match_status=FOUND_*/NOT_FOUND
      │  (4) wb_pass.py / alibaba_pass.py — добор цен (Wildberries ₽, Alibaba опт-$)
      │       fx_rate.py — живые курсы Нацбанка РК
      ▼
      │  (5) publish.py  — расчёт маржи, публикация
      ▼
   lots (БД, витрина только читает)
      │
      ▼
   app.py (FastAPI)  →  витрина (/), кабинет, аналитика, админ-панель (/admin)
```

**Определение «живой / закрытый» лот** — по факту присутствия в активной выдаче goszakup (статус 240 «приём ценовых предложений»), а **НЕ по дате дедлайна**: у переобъявленных лотов (ЗЦП2/ЗЦП3…) на странице висит срок старого этапа, поэтому дате верить нельзя. Источник истины — сам goszakup.

## Веб-приложение

Единое FastAPI-приложение (`web/app.py`) обслуживает и API, и страницы. Фронтенд — статические HTML со встроенным ванильным JS; графики и спарклайны рисуются вручную в SVG (без сторонних библиотек).

Страницы:

| Маршрут | Файл | Назначение |
|---------|------|------------|
| `/` | `index.html` | Витрина тендеров (публичная, read-only) |
| `/login` | `login.html` | Вход / регистрация |
| `/pricing` | `pricing.html` | Тарифы (конверсионная страница) |
| `/favorites` | `favorites.html` | Избранное — корзина сделок + мини-CRM |
| `/cabinet` | `cabinet.html` | Личный кабинет клиента с цифрами |
| `/analytics` | `analytics.html` | Аналитика рынка (категории + сезонность) |
| `/admin` | `admin.html` | Панель управления (владелец / админ-пароль) |

### Витрина (клиентская часть)

Светлая тема с бирюзовым акцентом. По каждому лоту карточка показывает: название и категорию, регион поставки, найденный товар (бренд/модель), **цену лота** (бюджет заказчика — публично) и **цену закупки + маржу** (платно). Поддержаны поиск, фильтры по статусу/категории/региону, сортировки (по уверенности, повторяемости, новизне, марже, срокам).

### Авторизация и доступ

- Регистрация по email + пароль (хеш **scrypt**), серверные сессии (токен хранится в `localStorage`, отправляется заголовком `X-Auth-Token`).
- Роли: `owner` (полный доступ + админка) и `client`.
- **14 дней полного доступа** при регистрации (триал), затем — демо-режим.

### Монетизация: тарифы и гейтинг

Доступ к «ценным» полям решает функция `_entitlement(user)`:

- **Открыто** (entitled): владелец · платный тариф (не истёкший) · триал (не истёкший).
- **Скрыто**: демо · гость · истёкшая подписка.

Скрываемые поля (`_gate_row`): `purchase_price`, `margin`, `margin_pct`, `margin_total`, `source_url`, `candidates`. На демо они приходят как `null`, а в интерфейсе показывается состояние «открыть доступ». Цена лота, регион, заказчик, дедлайн и повторяемость — публичны (это маркетинговый крючок).

Параметры: `TRIAL_DAYS = 14`, `FREE_LOT_LIMIT` — лимит карточек для незалогиненных.

### Инструменты клиента

Построены поверх избранного и карточки лота:

- **Умные фильтры по деньгам** — «бюджет на закуп ≤ X» и «заработок ≥ Y».
- **Чипы дедлайнов** — ⏰ горит (≤3 дн.), скоро (≤7 дн.), завершён.
- **Избранное — корзина сделок** — по каждому тендеру закуп и маржа, плюс суммарные «сколько уходит» и «сколько заработаешь».
- **Алерты по дедлайнам** избранного (на витрине и в избранном).
- **Калькулятор ставки** — ползунок цены от закупа до потолка, живой расчёт прибыли/маржи/недоцена, режим «хочу заработать чисто N». Учитывает **реальные расходы** (налог, комиссия площадки, доставка, обеспечение) — «прибыль в кармане».
- **Мини-CRM сделок** — стадии `интересно → подал заявку → выиграл → исполняю → проиграл`, фильтр по стадии, метрика «выиграно».
- **Заметки и чек-лист участия** на каждый тендер (стандартные шаги под goszakup: ЭЦП, документы, расчёт цены, обеспечение, подача) с автосохранением.
- **Фильтр по региону** — регион определяется из заказчика (SQL-функция `tf_region`, все 17 регионов РК).
- **Повторяющийся спрос** — «товар нужен в N тендерах»; клик показывает все эти тендеры (закупил раз — участвуй во многих).
- **Несколько поставщиков на лот** — все площадки, где найден товар, со ссылками (главный помечен «выбран»).
- **Инфо о заказчике** — клик по заказчику открывает его профиль (число тендеров, суммарный бюджет, регион, топ категории) и фильтрует витрину на его тендеры.

### Аналитика

- **Личный кабинет** (`/cabinet`) — персональный дашборд: воронка сделок, потенциальная и выигранная прибыль, конверсия, горящие сроки, готовность к подаче, оборотный капитал, статус подписки.
- **Аналитика рынка** (`/analytics`) — рейтинг категорий по марже / объёму (с потенциалом ниши) и сезонность тендеров по месяцам срока подачи. Маржа в рейтинге — платная.

### Админ-панель

Тёмная панель управления (`/admin`):

- **Здоровье пайплайна** — воронка стадий, покрытие (сколько лотов с ценой/маржой).
- **Мониторинг воркеров** — что и когда отрабатывало.
- **Просмотр лотов** — с фильтрами и карточкой лота.
- **Управление клиентами** — блокировка/разблокировка, назначение тарифа (с записью платежа).
- **Бизнес-метрики** — MRR (годовые ÷ 12), ARR, ARPU, churn, LTV, конверсия, доход, рост, разбивка по тарифам, лента платежей.
- **Тренды** — история по клиентам, доходу, витрине, марже, throughput (накапливается в `metrics_daily`).

## Модель данных

PostgreSQL. Две ключевые таблицы пайплайна + таблицы веб-приложения.

**`tenders`** — сырой источник с goszakup (пишет пайплайн):

| Поле | Описание |
|------|----------|
| `id` (PK) | внутренний id |
| `lot_number` (UNIQUE) | номер лота goszakup |
| `name`, `price_per_unit`, `quantity`, `unit` | предмет, цена за единицу, количество |
| `customer` | заказчик (из него выводится регион) |
| `deadline` | срок приёма заявок |
| `match_status` | `FOUND_EXACT` / `FOUND_PARTIAL` / `NOT_FOUND` |
| `match_result` (jsonb) | результат сверки: цена, площадка, ссылка, **кандидаты** (все найденные площадки) |
| `is_closed`, `last_seen`, `stage`, `collected_at` | жизненный цикл и стадия обработки |

**`lots`** — опубликованная витрина (читает API; `row_id` = `tenders.id`):

| Поле | Описание |
|------|----------|
| `row_id` (PK) | = id исходного тендера |
| `name`, `category`, `quantity`, `unit`, `customer` | предмет лота |
| `found_brand`, `found_model`, `found_product` | найденный товар |
| `source_url`, `source_site`, `confidence` | площадка-источник и уверенность сверки |
| `lot_price` | цена/потолок лота (**публично**) |
| `purchase_price`, `margin`, `margin_pct`, `margin_total` | цена закупки и маржа (**гейтится**) |
| `matched_specs`, `missing_specs`, `conflicts` (jsonb) | разбор соответствия ТЗ |
| `updated_at` | время публикации |

**`users`** — `id`, `email` (UNIQUE), `password_hash` (scrypt), `role`, `status`, `plan`, `plan_expires_at`, `is_trial`.
**`sessions`** — `token_hash` (PK), `user_id` (FK), `expires_at`.
**`payments`** — `id`, `user_id` (FK), `plan`, `period`, `amount`, `method`, `status`, `created_at`.
**`metrics_daily`** — `day` (PK), `users_total`, `paid`, `live`, `found`, `published`, `priced`, `avg_margin` (снимки для трендов).
**`favorites`** — `(user_id, lot_id)` (PK), `status` (стадия сделки), `note`, `checklist` (jsonb), `created_at`.

Производные «на лету» в API: `tf_region(customer)` — регион из заказчика; `demand_count` — оконный подсчёт повторяемости товара (бренд+модель).

## API

Все ответы — JSON. Клиентские эндпоинты требуют заголовок `X-Auth-Token`, админские — `X-Admin-Token` (или роль владельца).

**Витрина (публично, с гейтингом по доступу):**

```
GET  /api/lots                 # лоты с фильтрами: q, status, category, region,
                               #   customer, dbrand+dmodel (повторяемость),
                               #   max_spend, min_margin, sort, limit, offset
GET  /api/lots/{row_id}        # карточка лота
GET  /api/customer?name=...    # профиль заказчика (тендеры, бюджет, регион, категории)
GET  /api/regions              # регионы со счётчиками (для фильтра)
GET  /api/insights             # рейтинг категорий (маржа гейтится) + сезонность
GET  /api/stats                # сводка витрины (статусы, ср. уверенность, категории)
GET  /api/plans                # тарифы и активная промо-акция
GET  /healthz                  # проверка живости
```

**Авторизация:**

```
POST /api/auth/register        # email + пароль → сессия + триал 14 дней
POST /api/auth/login
POST /api/auth/logout
GET  /api/auth/me
POST /api/auth/change-password
```

**Избранное и сделки (клиент):**

```
GET    /api/favorites              # корзина: закуп/маржа по каждому + итоги + воронка
GET    /api/favorites/ids          # id избранного (для подсветки звёзд)
GET    /api/favorites/alerts       # дедлайны избранного (горящие/скоро)
POST   /api/favorites/{row_id}     # добавить
DELETE /api/favorites/{row_id}     # убрать
POST   /api/favorites/{row_id}/status     # стадия сделки
POST   /api/favorites/{row_id}/note       # заметка
POST   /api/favorites/{row_id}/checklist  # чек-лист подготовки
```

**Кабинет:**

```
GET  /api/cabinet              # воронка, прибыль, конверсия, дедлайны, подписка
```

**Админка (владелец / админ-пароль):**

```
GET  /api/admin/health                 # воронка стадий + покрытие
GET  /api/admin/workers                # статус воркеров
GET  /api/admin/lots                   # лоты с фильтрами
GET  /api/admin/lot/{lot_number}       # карточка лота (админ)
GET  /api/admin/lot-filter-options     # значения для фильтров
GET  /api/admin/clients                # клиенты
POST /api/admin/client/{uid}/block     # заблокировать
POST /api/admin/client/{uid}/unblock   # разблокировать
POST /api/admin/client/{uid}/plan      # назначить тариф (+ запись платежа)
GET  /api/admin/business               # MRR/ARR/ARPU/churn/LTV/доход/рост
GET  /api/admin/trends                 # тренды из metrics_daily
```

## Тарифы

| Тариф | Цена (мес / год) | Цена основателя | Для кого |
|-------|------------------|------------------|----------|
| **Демо** | 0 | — | Осмотреться (14 дней полного доступа, дальше цена/маржа скрыты) |
| **Старт** | 14 900 / 149 000 ₸ | 8 900 / 89 000 ₸ | Частным поставщикам и ИП |
| **Бизнес** | 39 900 / 399 000 ₸ | 24 900 / 249 000 ₸ | Отделам закупок и компаниям |
| **Под ключ** | по запросу | — | Интеграции и крупные клиенты |

Промо-акция «**Цена основателя**» — ограниченное число мест со скидкой; при назначении тарифа в этот период платёж записывается по founding-цене. Приём оплаты пока ручной (через админку); подключение Kaspi-эквайринга — в планах (тогда MRR/доход заполняются автоматически).

## Запуск и деплой

Полная пошаговая инструкция — в **[DEPLOY.md](DEPLOY.md)**. Коротко.

**Боевой деплой** (на сервере, после `git pull`):

```bash
cd /opt/tenderview/tender-pipeline
git reset --hard HEAD && git pull
bash deploy/sync.sh        # разложит web/, loops/, deploy/ по местам и пересоберёт app
```

`sync.sh` копирует `web/` (включая все `static/*.html` по маске — новые страницы подхватываются автоматически), `deploy/` и `loops/`, затем пересобирает контейнер приложения. Воркеры в `src/` обновляются самим `git pull`. Файл `.env` не трогается.

**Изменения только переменных окружения:**

```bash
cd /opt/tenderview
docker compose up -d --force-recreate app
```

Контейнеры: `tenderview-app-1` (FastAPI), `tenderview-db-1` (PostgreSQL). Caddy проксирует домен на `app:8000` (порт 8000 наружу не открыт).

## Статус и дорожная карта

**Готово и работает:** сбор и нормализация тендеров, поиск и сверка товара, расчёт маржи, публикация; авторизация и триал; монетизация с гейтингом; витрина с умными фильтрами; калькулятор ставки с реальными расходами; мини-CRM (статусы/заметки/чек-лист); фильтры по региону/спросу/заказчику; несколько поставщиков на лот; личный кабинет; аналитика рынка; админка с бизнес-метриками и трендами.

**В планах:**

- **Telegram-уведомления** — «ключевое слово + маржа» и дайджест (ключевое для удержания).
- **Kaspi-эквайринг** — автоматический приём оплаты (тогда MRR/доход заполняются сами).
- **Разбор NOT_FOUND** — ослабление верификатора LLM, чтобы поднять долю найденных лотов.
- **Прямой сбор региона/КАТО** с goszakup (сейчас регион выводится из заказчика эвристически).
